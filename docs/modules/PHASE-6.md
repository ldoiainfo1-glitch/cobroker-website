## Overview

Phase 6 handles the financial and legal layer of co-broking. Brokers need to manage deal documents (NDA, LOI, Agreement), collect brokerage payments, split commissions, and generate GST-compliant invoices.

**Input:** Completed Phase 4 (deals with co-broking agreements)  
**Output:** Document management, Razorpay payments, GST invoice generation

### Payment Flows

**1. Subscription (Recurring)**
```
User selects plan → Razorpay checkout →
Payment captured → Subscription activated →
Invoice generated → Email sent
```

**2. Commission Split (Between Brokers)**
```
Deal reaches "Completed" stage →
Platform calculates commission from deal value →
Both brokers see their share →
Broker A pays Broker B's share (or direct client payment) →
Invoice generated for each broker
```

### Razorpay Integration

```typescript
// Create order
const order = await razorpay.orders.create({
  amount: plan.price,     // in paise
  currency: 'INR',
  receipt: `sub_${Date.now()}`,
  notes: {
    companyId: company.id,
    planId: plan.id,
  },
});

// Verify payment (HMAC SHA256)
const signature = crypto
  .createHmac('sha256', process.env.RAZORPAY_KEY_SECRET)
  .update(`${orderId}|${paymentId}`)
  .digest('hex');

if (signature !== receivedSignature) {
  throw new Error('Payment verification failed');
}
```

## Module 21 — Invoicing

### Invoice Types

| Type | Description | Auto-generated? |
|------|-------------|----------------|
| Subscription Invoice | Monthly/annual subscription payment | Yes |
| Commission Invoice | Brokerage commission from client | Manual |
| Co-broking Split Invoice | Invoice between Broker A and B | Semi-auto |
| Receipt | Payment received confirmation | Yes |

### GST Compliance

- All invoices must include: GSTIN, HSN/SAC code, GST rate (18% for services)
- Invoice numbering: `CBK/2024-25/0001` (sequential, financial year)
- Company GSTIN must be verified before invoice generation
